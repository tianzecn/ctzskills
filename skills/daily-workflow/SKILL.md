# 日报工作流 (Daily Workflow)

## 📚 功能概述

这是一个统一的日报工作流程技能，整合了日记管理的所有常用功能。它是 `daily-efficiency-planner` Agent 的轻量级版本，适合需要快速执行单一或组合功能的场景。

## 🛡️ 关键操作检查点（必读）

执行以下任一操作前，**必须先列出清单 → 等待用户明确回复"确认"/"是"/"继续"，未确认前不得继续**：

| # | 触发场景 | 必须展示的清单 | 确认提示话术示例 |
|---|---------|--------------|----------------|
| C1 | **覆盖已存在的日记文件**（当天 `YYYY-MM-DD.md` 已存在） | 现有文件路径 + 文件最后修改时间 + 现有正文摘要（前 5 行） | "检测到当天日记已存在（最后修改：HH:MM）。是否覆盖？回复『覆盖』继续，否则跳过创建。" |
| C2 | **从源日记删除已同步任务**（任务同步后清理原文件） | 即将被删除的任务清单（含原文件路径 + 任务原文 + 行号） | "以上 N 个任务将从源文件删除（仅删除已成功同步到今日的部分）。回复『确认删除』继续，回复『保留』则只同步不删除。" |
| C3 | **过滤后任务清单为空** | 扫描范围 + 被过滤掉的全部任务原文 + 过滤命中的规则 | "前 3 天扫描到 N 个任务，全部被过滤规则排除。是否需要：(a) 放宽过滤规则重新同步 (b) 跳过同步直接创建空白日记？" |
| C4 | **跨月跨年的目录创建**（需要创建新月份/新年份目录） | 即将创建的完整目录路径 | "将创建跨[月/年]目录：path。回复『创建』继续。" |
| C5 | **任务优先级人工复核**（基于关键词的自动判断结果） | 自动判定为 P1 的任务清单 | （非阻塞）"以上自动判定为 P1，如需调整请直接告知，否则继续。" |

**默认安全原则**：
- 任何"删除"/"覆盖"动作默认 **不执行**，等待显式确认
- 自动同步任务只**追加**到当天日记，**不删除**源文件中的任务，除非用户单独要求清理
- 用户的回复必须含明确动词（"确认"/"覆盖"/"删除"），模糊回复（"嗯"、"好的"）按未确认处理

## 📐 执行流程（Step-by-Step）

无论触发的是单一模块还是组合模块，统一按下面 6 个 Phase 顺序执行。每个 Phase 内的 Step 编号唯一可追溯：

### Phase A · 输入解析（必跑）

- **Step A1**　调用 `get-current-datetime` 获取真实日期时间，**不得**用聊天上下文或训练数据中的日期。
- **Step A2**　基于触发词分类用户意图 → 决定要跑哪些模块（模块1 / 模块2 / 模块3，可组合）。
- **Step A3**　从当前时间推导路径变量 `{YY}`、`{YYMM}`、`{YYYY-MM-DD}` 及前 3 日清单。

### Phase B · 读盘扫描（仅模块1/2 触发）

- **Step B1**　按 `{YY}/{YYMM}/{YYYY-MM-DD}/{YYYY-MM-DD}.md` 逐文件 `Read`；文件不存在 → 静默跳过（依配置 `on_missing_source: skip`），**不得**报错或创建占位。
- **Step B2**　正则提取 `^\s*-\s*\[ \]\s*(.+)$` 抓出所有 `[ ]` 未完成任务，保留原文件路径 + 行号用于后续 C2 检查点。
- **Step B3**　跨月跨年路径需要新建目录时 → 触发 **C4 检查点**，等用户确认。

### Phase C · 过滤分类（仅模块2/3 触发）

- **Step C1**　按 "🔍 任务过滤规则" 的包含/排除模式过滤；命中排除的任务记录到「被过滤清单」备查。
- **Step C2**　过滤后剩余任务为零 → 触发 **C3 检查点**，列出被过滤清单 + 命中规则，等用户选放宽或跳过。
- **Step C3**　按 "优先级判断" 关键词把每个任务打 P1/P2/P3/P4 标签；记录所有 P1 任务用于 Step F2 的 C5 提示。

### Phase D · 检查点决策（高风险动作必跑）

- **Step D1**　如目标日记 `{YYYY-MM-DD}.md` 已存在 → 触发 **C1 检查点**：展示现有路径 + 修改时间 + 前 5 行摘要，等用户回复明确动词（"覆盖" / "跳过"）。
- **Step D2**　如用户显式要求清理源文件 → 触发 **C2 检查点**：列出待删任务清单（含原文件路径 + 任务原文 + 行号），等"确认删除" / "保留"。
- **Step D3**　所有 D 阶段决策结果**必须记录到本次回话上下文**，供 Phase E 使用；未确认前 Phase E 不得执行。

### Phase E · 写入执行（按 D 阶段决策结果）

- **Step E1**　按 "📋 日记文件模板" 生成主日记内容（Phase C 的任务按优先级分组写入「未完成计划」章节）。
- **Step E2**　仅在 D1 用户回复"覆盖"时才覆盖；否则 `Write` 新文件 / 追加到已有文件。**任何情况下都不得静默 overwrite**。
- **Step E3**　仅在 D2 用户回复"确认删除"时才回到源文件删除已同步任务；否则只追加不删源。
- **Step E4**　模块3 触发时：基于 Phase C 的任务清单 + "⏰ 能量-时间匹配策略" 生成时间块安排。

### Phase F · 输出反馈（必跑）

- **Step F1**　按 "📊 输出示例" 的结构组装回复：✅ 已创建/同步项 → 📋 任务清单（按优先级） → ⏰ 时间安排（如有） → 💡 效率建议。
- **Step F2**　末尾以 **C5 提示**（非阻塞）告知"以上 N 个任务自动判定为 P1"，方便用户人工复核。
- **Step F3**　**不得**生成 `{YYYY-MM-DD}-工作计划.md` 等附加文件（那是 daily-efficiency-planner Agent 的职责）。

> 💡 **回滚原则**：任意 Step 失败或用户拒绝某检查点 → 立即停止后续 Step，回报已完成项 + 阻塞原因，**不得**绕过检查点继续推进。

## 🎯 核心功能模块

### 模块 1: 日记启动 (Diary Starter)
**触发词**: "开始今天的日记"、"/today"、"新建日记"、"创建今日日记"

**功能**:
- ✅ 创建当天目录结构 `YY/YYMM/YYYY-MM-DD/`（跨月/跨年时触发 **C4 检查点**）
- ✅ 生成标准格式的主日记文件 `YYYY-MM-DD.md`（已存在时触发 **C1 检查点**，不静默覆盖）
- ✅ 自动同步前3天未完成任务到"未完成计划"章节（仅追加，不删源；过滤后为空触发 **C3 检查点**）
- ✅ 初始化日记基本框架（今日新增、亮点、想法、总结）

### 模块 2: 任务同步 (Task Sync)
**触发词**: "同步任务"、"查看未完成任务"、"任务清单"、"有哪些待办"

**功能**:
- ✅ 扫描前3天日记的 `[ ]` 未完成任务（日记文件不存在时静默跳过，不报错）
- ✅ 智能过滤（保留工作、学习、项目；排除临时、情感类）；过滤后为零任务触发 **C3 检查点**
- ✅ 按优先级分类（P1-P4）；自动判定为 P1 的任务列出后触发 **C5 提示**（非阻塞）
- ✅ 以简洁列表形式展示
- ⚠️ **默认仅追加到今日日记，不删除源文件中的任务**。若用户额外要求清理源文件，必须先列出待删清单 → 触发 **C2 检查点** → 确认后才删除

### 模块 3: 时间规划 (Time Planning)
**触发词**: "帮我规划今天"、"制定时间安排"、"优化日程"、"时间安排建议"

**功能**:
- ✅ 分析任务列表和预估时长
- ✅ 匹配能量-时间段（高能量时段做重要任务）
- ✅ 检测时间冲突并调整
- ✅ 生成优化的时间块安排
- ✅ 提供效率提升建议

## 🔄 智能组合执行

根据用户请求，自动组合执行多个模块：

| 用户请求 | 执行模块 |
|---------|---------|
| "开始今天的日记" | 模块1 + 模块2 |
| "同步任务并规划时间" | 模块2 + 模块3 |
| "完整启动今天的工作" | 模块1 + 模块2 + 模块3 |
| "只查看未完成任务" | 模块2 |
| "只规划时间" | 模块3 |

## 🔍 任务过滤规则

### 包含模式（会被同步）
- **工作相关**: 重要.* 、项目.* 、会议.* 、目标.*
- **学习成长**: 学习.* 、准备.* 、完成.*
- **业务发展**: 电商.* 、商标.* 、文化.* 、运营.*

### 排除模式（会被过滤）
- **临时事件**: 今天.* 、昨天.* 、临时.* 、随便.*
- **情感记录**: 情感.* 、感受.* 、想想.* 、可能.*

### 优先级判断

**P1 (🔴 紧急重要)**
- 关键词: 紧急、重要、立即、今天必须、deadline、客户、合同
- 立即处理，优先安排在高能量时段

**P2 (🟡 重要不紧急)**
- 关键词: 项目、学习、目标、准备、规划、设计
- 核心工作时间处理

**P3 (🔵 紧急不重要)**
- 关键词: 联系、确认、询问、查看、检查
- 委托或快速处理

**P4 (⚪ 不紧急不重要)**
- 关键词: 整理、归档、阅读、了解、考虑
- 可推迟或删除

## ⏰ 能量-时间匹配策略

| 能量段 | 时间窗口 | 适配任务类型 | 时间块配置 |
|-------|---------|------------|-----------|
| 🌅 **高** | 08:00-11:00 / 14:00-16:00 | 创意/战略/深度学习/核心项目 | 深度块 90min + 15min 休息，每日最多 3 块 |
| ⚡ **中** | 11:00-12:00 / 16:00-18:00 | 会议/邮件/协调/日常业务 | 专注块 45min + 10min 休息，每日最多 6 块 |
| 🌙 **低** | 07:00-08:00 / 18:00-21:00 | 整理/重复任务/阅读/日程回顾 | 番茄块 25min + 5min 休息 |

## 🔧 配置参数

```yaml
# 任务同步配置
task_sync:
  scan_days: 3                # 扫描前几天的日记
  min_priority: P4            # 显示的最低优先级
  auto_remove_synced: false   # 默认 false — 不删除源文件任务（避免误删）
  require_confirmation: true  # 任何删除/覆盖动作必须用户确认（C1/C2 检查点）
  on_missing_source: skip     # 源日记不存在时静默跳过该日，不抛错
  on_empty_after_filter: ask  # 过滤后为零任务时触发 C3 检查点

# 时间规划配置
time_planning:
  working_hours:
    start: "08:00"
    end: "18:00"
  deep_work_limit: 3        # 每日深度工作块上限
  buffer_time: 10           # 任务间缓冲时间（分钟）

# 能量时段配置
energy_levels:
  high: ["08:00-11:00", "14:00-16:00"]
  medium: ["11:00-12:00", "16:00-18:00"]
  low: ["07:00-08:00", "18:00-21:00"]
```

## ⚠️ 注意事项

1. **文件冲突处理**: 当天日记已存在时触发 **C1 检查点**，不静默覆盖，等待用户回复"覆盖"才执行
2. **跨月跨年**: 正确处理日期计算，需要创建新月份/新年份目录时触发 **C4 检查点**
3. **优先级判断**: 基于关键词自动判断，自动判定为 P1 的任务通过 **C5 提示**透明告知用户（非阻塞）
4. **时间预留**: 自动预留20%时间应对突发情况
5. **任务清理安全**: 默认 `auto_remove_synced: false`，仅追加不删除；用户显式要求清理时触发 **C2 检查点**
6. **源文件不可读**: 前3天日记文件不存在时静默跳过该日，不报错也不创建占位

## 🆚 与 Agent 的边界

此 Skill 仅做轻量级日记创建/任务同步/时间规划。**不**生成 `YYYY-MM-DD-工作计划.md`、**不**推送飞书、**不**做周/月效率报告、**不**做全天进度监控、**不**做晚间总结——以上交给 `daily-efficiency-planner` Agent。两者职责互补，不可越权（参见 Step F3）。
